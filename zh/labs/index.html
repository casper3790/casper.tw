01

Home Lab

內容
建立一個可以反覆部署、測試、失敗與復原的個人實驗環境。
原因
把網路、系統與安全概念放進真實環境裡驗證，而不是只停在閱讀或想像。
下一步
整理基礎拓樸、服務分層與紀錄方式，讓每一次實驗都能被回顧。

02

ESP32 / M5Stack

內容
用小型裝置測試感測、通訊、資料輸入與邊緣裝置行為。
原因
理解嵌入式設備如何產生資料，也觀察資料在進入系統前可能出現的風險。
下一步
建立更穩定的資料傳輸與驗證流程，記錄失敗案例。
查看實驗 →
03

ROS2 Robotics

內容
驗證 ROS2 topic、節點通訊、感測資料與機器人控制流程。
原因
把軟體、硬體與環境不確定性放在一起觀察，理解自主系統如何被建構。
下一步
持續拆解 publish / subscribe、感測整合與控制可靠性。
查看實驗 →
04

PLC / Industrial Security

內容
探索工控流程、PLC 邏輯與工業環境中的安全邊界。
原因
工業系統的風險常藏在流程、信任與操作習慣之間，需要用實驗慢慢釐清。
下一步
從基礎控制邏輯與通訊觀察開始，建立可重複驗證的筆記。
