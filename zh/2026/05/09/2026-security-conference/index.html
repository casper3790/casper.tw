2026 資安大會參訪心得：從展場看見資安產業的真實樣貌

2026-05-09 11 分鐘閱讀

2026 資安大會參訪心得：從展場看見資安產業的真實樣貌這次參加 2026 資安大會，我花了三天時間觀察展場、聽議題、看不同廠商展示資安解決方案。

2026 資安大會參訪心得：從展場看見資安產業的真實樣貌

這次參加 2026 資安大會，我花了三天時間觀察展場、聽議題、看不同廠商展示資安解決方案。
對我來說，這不只是一次參訪活動，而是一次重新理解「資安產業到底在解決什麼問題」的機會。

以前我對資安的想像，可能比較集中在 CTF、滲透測試、漏洞利用、網站攻防這些比較技術導向的內容。
但這次走完整個展場後，我發現真正的資安產業其實非常廣，從人員教育、企業治理、雲端防護、AI 安全、OT 工控資安，到醫療、車載、通訊與量子資安，都已經成為資安的重要場景。

一、零信任架構：不再預設任何人是安全的

這次展場中，我看到很多廠商都在強調「零信任架構」。

零信任的核心概念是：

不因為使用者在公司內網，就直接相信他是安全的。

每一次存取都要經過驗證、授權與監控。
這讓我重新理解到，現代資安已經不只是防火牆擋外部攻擊，而是要假設攻擊者可能已經在內部，所以每一個帳號、裝置、系統與資料存取都必須被確認。

我覺得零信任架構很適合拿來跟學校、企業、實驗室環境結合，因為現在很多攻擊不是從正面突破，而是透過帳號、釣魚信、權限濫用或內部裝置開始擴散。

二、社交工程演練與人員資安：人仍然是最重要的防線

這次我也看到很多與社交工程演練、人員資安相關的內容。

社交工程不是單純的「騙人」，而是利用人的信任、習慣與疏忽來繞過技術防線。例如：

  • 釣魚信件
  • 假登入頁面
  • 惡意附件
  • 假冒主管或廠商
  • 透過通訊軟體誘導點擊連結

這讓我發現，資安不只是設備與系統的問題，也是教育與行為管理的問題。
即使公司買了很貴的資安設備，如果使用者沒有安全意識，還是可能因為一封 Email 或一個假網站造成重大資安事件。

所以我覺得未來資安教育應該要更生活化，不只是教規則，而是要讓使用者真的理解攻擊者會怎麼利用人的心理。

三、ROS 無人機器人資安：機器人也需要資安防護

我這次特別注意到 ROS、無人機與機器人相關的資安議題。

因為機器人系統通常會結合：

  • 感測器
  • 攝影機
  • LiDAR
  • 無線通訊
  • 控制系統
  • 邊緣運算裝置
  • ROS / ROS2 架構

這些系統如果沒有做好資安防護，可能會被攻擊者攔截資料、竄改控制指令，甚至影響機器人的行動。

這讓我想到，未來資安不只會存在於網站與伺服器，也會進入機器人、自動化設備、智慧工廠與無人載具。
尤其是 ROS 這類開放式機器人架構，如果部署在真實場域，就需要考慮通訊安全、權限控管、節點驗證與異常行為偵測。

這個方向也讓我覺得，資安與機器人其實可以結合成很有發展性的專題。

四、大語言模型安全護欄：AI 時代的新攻防問題

這次展場中，大語言模型的安全護欄也是很熱門的主題。

現在很多企業開始導入 AI 助理、客服機器人、文件分析系統與內部知識庫問答系統。
但只要 AI 可以讀取資料、產生回應，就會出現新的安全問題，例如：

  • Prompt Injection
  • 越權讀取內部資料
  • 敏感資訊外洩
  • AI 產生錯誤建議
  • 使用者誘導模型繞過限制
  • 模型回應不符合公司政策

所以 AI 安全護欄的目標，不只是讓 AI 不亂講話，而是要確保 AI 在使用企業資料時，不會洩漏機密、不會違反權限，也不會被惡意提示詞操控。

我覺得這是未來資安非常重要的方向，因為 AI 會越來越像企業內部的新入口。
以前攻擊者攻擊網站、API、帳號，未來也可能攻擊 AI 助理。

五、雲端資安、API 安全與機密管理

這次也看到很多雲端資安與 API 安全相關展示。

現代企業大量使用雲端服務，資料不一定只放在自己的機房，而是分散在不同的雲平台、SaaS 服務與 API 系統中。
這代表資安防護也必須跟著改變。

我觀察到幾個重要方向：

  • 雲端帳號權限控管
  • API 存取驗證
  • 金鑰與 Token 管理
  • 機密資料保護
  • 雲端環境設定檢查
  • 異常存取偵測

其中我覺得「機密管理」非常重要，因為很多資安事件不是系統被直接打穿，而是 API Key、Token、憑證或密碼被放在錯誤的位置，例如 GitHub、設定檔或公開伺服器。

這也提醒我，以後做網站、後端、AI API 或雲端部署時，不能只是讓功能跑起來，也要思考金鑰、權限與紀錄保存的問題。

六、檔案存取軌跡稽核與使用者行為監控

展場中也有許多關於資料存取紀錄、檔案軌跡稽核與使用者行為監控的方案。

這類技術的重點是：

當資料被讀取、複製、下載、分享或異常存取時，系統能不能留下證據並即時發現？

這對企業來說非常重要，因為很多資料外洩事件不是一瞬間發生，而是長時間慢慢累積。
如果系統能記錄誰在什麼時間、用什麼裝置、存取了哪些檔案，就能在事件發生後追查來源，也能在異常行為出現時提前警示。

這讓我理解到，藍隊防禦不是只看攻擊封包，也要看使用者行為、資料流向與存取模式。

七、印表機資安：容易被忽略的企業入口

我以前比較少注意印表機資安，但這次展場讓我發現，印表機其實也是企業資安的一部分。

企業印表機通常連接內網，可能具備掃描、列印、傳真、Email 寄送、檔案暫存與帳號登入功能。
如果印表機沒有更新、沒有設定權限、沒有加密傳輸，就可能成為攻擊者進入內網或取得文件的入口。

這讓我想到，資安不是只保護電腦和伺服器，也要保護所有連上網路的設備。
只要設備能連網、能存資料、能傳輸資料，就有可能成為攻擊面。

八、醫療資安：資安會直接影響人的安全

醫療資安是我這次很有感的主題之一。

醫療場域裡有很多系統與設備，例如：

  • 醫療資訊系統
  • 病歷系統
  • 醫療影像系統
  • 醫療 IoT
  • 生命監測設備
  • 醫院內部網路
  • 護理與行政系統

這些系統一旦遭受攻擊，不只是資料外洩，還可能影響醫療流程與病人安全。

這讓我意識到，醫療資安與一般企業資安不同，它不只是保護資料，而是保護醫療服務能不能正常運作。
未來如果我要做醫療資安專題，應該可以從「邊緣裝置異常偵測」、「醫療 AI 模型安全」或「醫療設備存取控管」這些方向切入。

九、通訊安全、防護與 SSL 憑證

通訊安全也是展場中很基礎但很重要的一塊。

不論是網站、API、企業系統、雲端服務還是內部平台，只要資料需要傳輸，就必須考慮：

  • 加密通訊
  • SSL / TLS 憑證
  • 憑證生命週期管理
  • 中間人攻擊防護
  • 內外部通訊驗證
  • 通訊紀錄稽核

SSL 憑證看起來只是網站上的小鎖頭，但實際上它代表的是通訊是否被加密、伺服器身分是否可信，以及使用者資料是否能被安全傳輸。

這也提醒我，以後做網站部署時，不能只在意畫面好不好看，也要注意 HTTPS、憑證更新與安全設定。

十、資安治理：技術之外的管理能力

這次我也看到很多資安治理相關內容。

資安治理比較不像 CTF 那樣有明確的題目與答案，它更像是在回答：

  • 公司要怎麼制定資安政策？
  • 發生事件時誰要負責？
  • 哪些資料需要被保護？
  • 權限要怎麼分級？
  • 風險要怎麼評估？
  • 資安預算要怎麼分配？
  • 如何符合稽核與法規要求？

這讓我理解到，真正的資安不是只有攻擊與防禦，也包含制度、流程、責任分工與長期管理。

一間公司如果只有工具，沒有治理，很容易變成買了很多產品但不知道怎麼用。
所以資安治理其實是讓技術真正落地的關鍵。

十一、AI SOC 與自主性資安防護：藍隊正在變得更自動化

這次我看到 AI SOC 與自主性資安防護的展示，讓我感覺藍隊正在進入新的階段。

傳統 SOC 需要大量人力去看告警、分析事件、判斷威脅來源。
但現在攻擊數量太多，單靠人工已經很難處理所有資訊，所以 AI SOC 開始被用來協助：

  • 自動整理告警
  • 判斷事件嚴重程度
  • 關聯不同系統的紀錄
  • 協助威脅狩獵
  • 產生事件報告
  • 提供處置建議

我覺得這對學生來說是一個很值得學的方向，因為未來的藍隊不只是看 log，而是要懂得如何使用 AI、SIEM、EDR、NDR、SOAR 等工具來進行整體防禦。

十二、AI 主動數位鑑識機制

數位鑑識也是這次讓我印象深刻的主題。

傳統數位鑑識通常是在事件發生後，才開始蒐集證據、分析紀錄、還原攻擊路徑。
但 AI 主動數位鑑識的概念，是希望在事件發生的過程中，就能自動蒐集關鍵證據，幫助防禦者更快了解發生什麼事。

這對資安事件處理很重要，因為很多攻擊痕跡如果沒有即時保存，事後可能會被覆蓋或刪除。

我認為未來的資安不只是「偵測攻擊」，還要能「保留證據」、「還原事件」與「輔助決策」。

十三、OT 資安、NOC 與關鍵基礎設施

OT 資安也是這次展場的重要主題之一。

OT 和一般 IT 不同，它更接近工廠、電力、交通、水資源、製造業與自動化控制系統。
這些系統如果出問題，可能不是單純網站不能用，而是會影響實體世界的運作。

我也看到 NOC 相關概念，NOC 比較偏向網路維運中心，負責監控網路穩定性、流量狀態與服務可用性。
而 SOC 則更偏向安全事件監控與威脅分析。

這讓我理解到，未來的資安人才可能不能只懂單一領域，而是要懂 IT、OT、網路、系統與安全營運之間的關係。

十四、電子郵件安全與攻擊入口

電子郵件安全也是展場中非常重要的一塊。

很多攻擊其實都是從 Email 開始，例如：

  • 釣魚信
  • 惡意附件
  • 假冒通知
  • 假發票
  • 假登入頁面
  • 商務郵件詐騙

這讓我覺得，Email 雖然是很傳統的工具，但它仍然是攻擊者最常使用的入口之一。
所以企業不只要有郵件過濾系統，也要有員工教育、通報流程與演練機制。

十五、車載資安：汽車正在變成會移動的電腦

車載資安也是我覺得很有未來感的主題。

現在的車子越來越像一台大型電腦，裡面有感測器、通訊模組、車用網路、娛樂系統、控制系統與雲端連線。
當汽車變得越來越智慧化，攻擊面也會變多。

車載資安不只是保護車上的資料，也可能牽涉到行車安全。
這讓我發現，未來資安會越來越貼近真實世界，不只是虛擬空間裡的攻防。

十六、量子資安：提前準備未來的密碼學挑戰

量子資安是這次展場中比較前瞻的主題。

雖然量子電腦還沒有全面普及，但它可能會對傳統密碼學造成影響。
所以現在很多單位開始討論後量子密碼學與量子安全通訊，希望在未來量子運算能力成熟前，先準備新的安全架構。

這讓我覺得資安是一個需要提前思考未來的領域。
不是等攻擊發生才補救，而是要預測未來可能出現的風險。

十七、我這三天最大的收穫

這三天參觀下來，我最大的收穫是：

資安不是單一技術，而是一整個保護數位社會運作的系統。

它包含攻擊、防禦、治理、教育、監控、鑑識、雲端、AI、醫療、工控、機器人、車載與通訊安全。

以前我可能會把資安想成「會不會打 CTF」、「會不會用 Kali」、「會不會找漏洞」。
但這次參訪讓我看到，真正的資安產業需要的不只是攻擊能力，也需要理解場域、系統、流程與人。

十八、對我未來方向的影響

這次資安大會讓我更確定，未來我想走的方向不只是單純的網站資安，而是希望把資安和我正在做的專題結合，例如：

  • 醫療資安
  • AI 安全
  • 機器人資安
  • ROS / 無人系統資安
  • 藍隊防禦
  • AI SOC
  • OT 資安
  • 數位鑑識
  • 雲端與 API 安全

我覺得這些方向都很適合延伸成未來的專題、研究或競賽作品。

尤其是「AI + 資安」、「醫療 + 資安」、「機器人 + 資安」這三個方向，對我來說非常有吸引力。
因為它們不只是技術展示，而是真的有機會解決現實世界中的安全問題。

結語

這次 2026 資安大會讓我看到資安產業的廣度，也讓我更清楚自己未來可以努力的方向。

資安不是只有攻擊，也不是只有防禦。
它是一種讓系統、資料、人員與社會能夠安全運作的能力。

未來我希望自己不只是學會工具，而是能夠真正理解不同場域的問題，並用資安技術去設計出有價值的解決方案。
