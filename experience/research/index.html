Research Directions

Questions worth spending years on.
01

Medical Security

Research Question
How can data, devices, and people inside a clinical workflow form a trustworthy validation relationship?
Why it matters
Medical security is not a single-system problem. Risk appears where clinical workflow, data input, device trust, and human judgment meet.
Current Direction
Start from data validation, device trust, and workflow risk, then turn the observations into research notes that can be traced and discussed.
02

Industrial Security

Research Question
How can the security boundary of an industrial control system be understood without interrupting the process?
Why it matters
Industrial environments depend on stability and continuity, so security work has to understand real operations instead of applying a generic IT model.
Current Direction
Observe PLC logic, control behavior, and communication flows, then build a path from basic experiments to risk analysis.
03

Communication Security

Research Question
When systems exchange data, how can they know that a message really came from a trusted source?
Why it matters
Many risks appear between transmission, validation, and trust boundaries. Communication security decides whether systems can cooperate reliably.
Current Direction
Begin with networking fundamentals, protocol behavior, and data validation, then document experiments that can be reproduced.
04

Security Education

Research Question
How can beginners understand security without being overwhelmed by frustration?
Why it matters
Security education is not only about tools. It should help people build problem awareness, validation habits, and a direction for continued learning.
Current Direction
Organize CTF, Web security, networking, and hands-on teaching experience into quieter learning material that can keep growing.
